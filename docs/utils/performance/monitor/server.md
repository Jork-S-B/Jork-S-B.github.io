## 📌 指标分析

### 🚁 系统平均负载

CPU在一段时间内的使用量

一般来说，系统平均负载超过`CPU核数`数倍时，视为异常。

#### 🔧 实时监控系统性能

* top -u user # -u指定用户启动的进程
* top -p pid # -u -p不能一起用

交互按键:

* 数字键1，切换显示每个CPU核心的使用情况。
* P，按CPU使用率排序
* M，按内存使用量排序
* u -> 输入用户名 -> 回车，查看某个用户的进程
* k -> 输入pid，杀进程
* R，反转当前排序。
* Z，切换颜色模式。

每个CPU核心使用情况:

* us - user 用户使用量
* sy - system 系统使用量
* idl - idle 空闲使用量
* wa - wait io等待使用量

load average - 平均负载，有3个数值，分别代表1、5、15分钟内的平均负载。

若数值递减，说明系统压力逐渐增大。若超值大于等于1，说明超负荷。
